## `mapOf()` как это использовать и стоит ли ?
1. Что такое `mapOf()` в Kotlin:

      **`mapOf()` неизменяемая** коллекция, где каждый элемент имеет имеет ключ и значение.

      После создания такую коллекцию нельзя изменить (добавить/удалить элементы или изменить существующие)

Пример:
```kotlin
      private fun digitButtons() {
        val digitButtons = mapOf(
            R.id.btn0 to "0",
            R.id.btn1 to "1",
            R.id.btn2 to "2",
            R.id.btn3 to "3",
            R.id.btn4 to "4",
            R.id.btn5 to "5",
            R.id.btn6 to "6",
            R.id.btn7 to "7",
            R.id.btn8 to "8",
            R.id.btn9 to "9"
        )
}
```
2. Формирование ключей в `mapOf()`:
   
     Уникальность ключей:
     Каждый ключ должен быть уникальным.

     Если попытаться создать map с дублирующимися ключами, получим исключение `IllegalArgumentException`

     Типы ключей: В качестве ключей можно использовать практически любые типы данных:

     Int (наиболее распространено в Android для R.id.*)

     String (для именованных констант)

     и *Т.Д*

4. Плюсы и минусы использования mapOf()

         Плюсы:

   Безопасность данных: Неизменяемость гарантирует, что данные не будут случайно изменены в других частях приложения

   Читаемость кода: Синтаксис ключ to значение делает код интуитивно понятным

   Типобезопасность: Компилятор Kotlin проверяет типы ключей и значений еще на этапе компиляции

   Эффективный поиск: Доступ к значению по ключу выполняется за O(1) в среднем случае, что обеспечивает высокую производительность.


         Минусы:

   Отсутствие возможности изменения: Нельзя добавить/удалить элементы после создания мапы. Для динамических данных лучше использовать mutableMapOf()

   Потребление памяти: Хеш-таблицы требуют больше памяти, чем простые массивы или списки

   Ограниченная функциональность поиска: Поддерживается только точное совпадение ключа. Нет встроенного поиска по префиксу, диапазону или регулярному выражению

   Отсутствие гарантии порядка: mapOf() не гарантирует сохранение порядка элементов при итерации (хотя на практике порядок вставки часто сохраняется). Для упорядоченных данных лучше использовать listOf() с парами.


5. Когда не стоит использовать `mapOf():`
   
   Если используются часто изменяемые данные

   При использовании больших объемов данных

   Когда важен порядок элементов
